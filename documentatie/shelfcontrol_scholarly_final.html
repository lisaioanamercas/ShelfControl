1.4 Scopul aplicației

Shelf Control este o aplicație web modernă care ajută utilizatorii să-și organizeze colecțiile de cărți într-un mod intuitiv și eficient. Aplicația oferă funcționalități complete pentru gestionarea bibliotecilor personale, de la catalogarea cărților până la urmărirea progresului de lectură.

Obiective principale:

  • Crearea unui spațiu digital intuitiv pentru gestionarea colecțiilor de cărți
  • Oferirea de instrumente avansate de căutare și filtrare
  • Generarea de statistici detaliate despre obiceiurile de lectură
  • Facilitarea descoperirii de noi titluri prin integrarea cu Google Books API
  • Crearea unei comunități de cititori prin funcționalități sociale

3. Arhitectura aplicației

Aplicația Shelf Control utilizează o arhitectură în trei straturi (3-tier) optimizată pentru gestionarea eficientă a colecțiilor de cărți. Arhitectura se bazează pe principiile de separare a responsabilităților și modularitate.

3.1 Straturi arhitecturale

  • Presentation Layer: HTML5, CSS3, JavaScript (Frontend)
  • Business Logic Layer: PHP cu funcții dedicate (Backend)
  • Data Access Layer: Oracle Database cu proceduri stocate

3.2 Componente principale

  • Authentication Service: Gestionarea JWT și sesiuni
  • Book Management Service: CRUD operații pentru cărți
  • User Management Service: Gestionarea conturilor și profilurilor
  • External API Integration: Conectori pentru Google Books și Turbo Pass
  • Statistics Engine: Generarea rapoartelor

Documentația API

6.1 Autentificare

Toate cererile către API necesită un token JWT valid în header:

Authorization: Bearer {jwt_token}

6.2 Endpoints principale

Utilizatori

POST /login
Body: {"email": "user@example.com", "password": "password"}
Response: {"token": "jwt_token", "user": {...}}
POST /register
Body: {"email": "user@example.com", "password": "password", "name": "Nume"}
Response: {"message": "User created successfully"}

Cărți

GET /filter-books
Query Params: ?title=titlu&genre=ficțiune
Response: {"books": [...], "total": 150, "pages": 8}

Colecții utilizator

POST /add-book?{book_id}
Body: {"status": "to_read|owned|read"}
Response: {"message": "Book added to collection"}

Securitate

Măsuri de securitate implementate

Autentificare și autorizare

  • JWT Tokens: Autentificare stateless cu expirare automată
  • Role-based Access: Separarea permisiunilor utilizator/admin
  • Password Hashing: Bcrypt cu salt pentru parolele utilizatorilor

Protecția datelor

  • SQL Injection Prevention: Bind parameters pentru toate query-urile

Troubleshooting

Probleme comune și soluții

Erori de autentificare

Problemă Cauză posibilă Soluție
Token-ul JWT a expirat Sesiunea a depășit timpul limită Re-autentificați-vă în aplicație
Login eșuat cu credențiale corecte Probleme de conectivitate cu baza de date Verificați statusul serverului Oracle
Cookie-urile nu se salvează Setări browser restrictive Activați cookie-urile pentru domeniu

Probleme cu căutarea cărților

Problemă Cauză posibilă Soluție
Rezultate limitate la căutare Limită API Google Books Folosiți termeni mai specifici
Căutarea nu returnează rezultate Cheie API invalidă Verificați configurația API
Încărcare lentă a imaginilor Conexiune internet lentă Implementați lazy loading

Probleme de performanță

  • Încărcare lentă: Verificați indexarea tabelelor în Oracle
  • Timeout-uri: Optimizați query-urile complexe
  • Memoria insuficientă: Creșteți limita PHP memory_limit

Limitări și restricții

9.1 Limitări tehnice

  • Google Books API: Maxim 1000 de cereri pe zi per cheie API
  • Dimensiune fișiere: Upload-ul de imagini limitat la 5MB
  • Concurrent users: Limitat de configurația serverului Oracle
  • Compatibilitate browser: Nu suportă Internet Explorer 11

9.2 Limitări funcționale

  • Limbă: Interface disponibilă doar în română
  • Backup: Backup-ul datelor se face manual
  • Notificări: Doar notificări în aplicație, fără email/SMS
  • Import/Export: Nu există funcționalitate de export masiv

9.3 Restricții de conținut

  • Recenzii: Limitate la 2000 de caractere
  • Grupuri: Maxim 50 de membri per grup
  • Cărți per utilizator: Maxim 10,000 de cărți în colecție
  • Imagini: Doar formate JPG, PNG, WebP acceptate
